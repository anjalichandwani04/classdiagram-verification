PROJECT: Word Processor
REQ:
A user can open a new or existing document. Text is entered through a keyboard. A document is made up of several pages and each page is made up of a header, body and footer. Date, time and page number may be added to header or footer. Document body is made up of sentences, which are themselves made up of words and punctuation characters. Words are made up of letters, digits and/or special characters. Pictures and tables may be inserted into the document body. Tables are made up of rows and columns and every cell in a table can contain both text and pictures. Users can save or print documents.


==================== Step 1: Identify Classes ====================
Based on the provided description, here are the main classes/entities identified, along with justifications for each:

1. **User**
   - Justification: Represents individuals who interact with the system to create, open, save, or print documents.

2. **Document**
   - Justification: Represents the main entity that users work with, which can be new or existing and contains multiple pages.

3. **Page**
   - Justification: Represents individual pages within a document, which contain structured content like headers, bodies, and footers.

4. **Header**
   - Justification: Represents the top section of a page, which may include metadata such as date, time, and page number.

5. **Body**
   - Justification: Represents the main content area of a page where text, images, and tables are placed.

6. **Footer**
   - Justification: Represents the bottom section of a page, which may also include metadata similar to the header.

7. **Sentence**
   - Justification: Represents a unit of text within the body, composed of words and punctuation characters.

8. **Word**
   - Justification: Represents individual units of text that make up sentences, consisting of letters, digits, and/or special characters.

9. **PunctuationCharacter**
   - Justification: Represents symbols used in sentences to structure and clarify meaning.

10. **Picture**
    - Justification: Represents visual elements that can be inserted into the document body.

11. **Table**
    - Justification: Represents structured data within the document body, consisting of rows and columns.

12. **Row**
    - Justification: Represents a horizontal grouping of cells within a table.

13. **Column**
    - Justification: Represents a vertical grouping of cells within a table.

14. **Cell**
    - Justification: Represents an individual unit within a table that can contain text and pictures.

15. **Character**
    - Justification: Represents the basic building blocks of words, which can include letters, digits, and special characters.

16. **Action**
    - Justification: Represents user actions such as opening, saving, or printing documents.

These classes encapsulate the key entities and concepts described in the text, forming a foundation for a UML model that reflects the structure and relationships within the document editing system.

==================== Step 3: Relationships ====================
Based on the provided description and the identified classes, here are the relationships between the classes, their types, and cardinalities:

1. **User** -- **Association** -- **Document** : 1..*  
   (A user can have multiple documents, but each document is associated with one user.)

2. **Document** -- **Composition** -- **Page** : 1..*  
   (A document is composed of multiple pages, and if the document is deleted, its pages are also deleted.)

3. **Page** -- **Composition** -- **Header** : 1..1  
   (Each page has exactly one header, and if the page is deleted, the header is also deleted.)

4. **Page** -- **Composition** -- **Body** : 1..1  
   (Each page has exactly one body, and if the page is deleted, the body is also deleted.)

5. **Page** -- **Composition** -- **Footer** : 1..1  
   (Each page has exactly one footer, and if the page is deleted, the footer is also deleted.)

6. **Body** -- **Composition** -- **Sentence** : 0..*  
   (A body can contain zero or more sentences, but a sentence must belong to a body.)

7. **Sentence** -- **Composition** -- **Word** : 0..*  
   (A sentence can contain zero or more words, but a word must belong to a sentence.)

8. **Sentence** -- **Composition** -- **PunctuationCharacter** : 0..*  
   (A sentence can contain zero or more punctuation characters, but a punctuation character must belong to a sentence.)

9. **Body** -- **Composition** -- **Picture** : 0..*  
   (A body can contain zero or more pictures, but a picture must belong to a body.)

10. **Body** -- **Composition** -- **Table** : 0..*  
    (A body can contain zero or more tables, but a table must belong to a body.)

11. **Table** -- **Composition** -- **Row** : 1..*  
    (A table is composed of one or more rows, and if the table is deleted, its rows are also deleted.)

12. **Table** -- **Composition** -- **Column** : 1..*  
    (A table is composed of one or more columns, and if the table is deleted, its columns are also deleted.)

13. **Row** -- **Composition** -- **Cell** : 1..*  
    (A row is composed of one or more cells, and if the row is deleted, its cells are also deleted.)

14. **Column** -- **Composition** -- **Cell** : 1..*  
    (A column is composed of one or more cells, and if the column is deleted, its cells are also deleted.)

15. **Cell** -- **Association** -- **Picture** : 0..1  
    (A cell can optionally contain one picture, but a picture can exist independently of a cell.)

16. **Cell** -- **Association** -- **Character** : 0..*  
    (A cell can contain zero or more characters, but a character must belong to a cell.)

17. **User** -- **Association** -- **Action** : 0..*  
    (A user can perform zero or more actions, but an action is associated with one user.)

This structure captures the relationships and cardinalities between the classes in the document editing system.
