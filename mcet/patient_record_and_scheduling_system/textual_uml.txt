CLASS: Patient
Description: Represents individuals receiving medical care. The system tracks their information, visit history, and treatment details.
ATTRIBUTES:
- patientID: String
- name: String
- dateOfBirth: Date
- gender: String
- contactInfo: String
- familyID: String
- primaryCarePhysicianID: String
METHODS:
- getPatientDetails(): Patient
- updateContactInfo(newContactInfo: String): void
- addVisit(visit: Visit): void
- getPatientHistory(): PatientHistory

CLASS: Family
Description: Represents a group of patients, with a head of family responsible for medical coverage. This entity helps in organizing patients under a common household.
ATTRIBUTES:
- familyID: String
- headOfFamilyID: String
- members: List<Patient>
METHODS:
- addMember(patient: Patient): void
- removeMember(patientID: String): void
- getFamilyMembers(): List<Patient>

CLASS: Doctor
Description: Represents medical professionals who provide care to patients. The system maintains information about doctors, including their roles as primary care physicians.
ATTRIBUTES:
- doctorID: String
- name: String
- specialization: String
- contactInfo: String
- patients: List<Patient>
METHODS:
- getDoctorDetails(): Doctor
- addPatient(patient: Patient): void
- removePatient(patientID: String): void
- viewPatientHistory(patientID: String): PatientHistory

CLASS: Receptionist
Description: Represents staff members who manage patient information and appointments. They are key users of the system for entering new patient data and scheduling.
ATTRIBUTES:
- receptionistID: String
- name: String
- contactInfo: String
METHODS:
- addNewPatient(patient: Patient): void
- scheduleAppointment(appointment: Appointment): void
- updatePatientInfo(patient: Patient): void

CLASS: Nurse
Description: Represents healthcare professionals who track patient visits, diagnoses, and medications. They play a crucial role in managing patient records and treatment information.
ATTRIBUTES:
- nurseID: String
- name: String
- contactInfo: String
METHODS:
- recordVisit(visit: Visit): void
- updateDiagnosis(visitID: String, diagnosis: Diagnosis): void
- prescribeMedication(visitID: String, medication: Medication): void
- printPatientHistory(patientID: String): void

CLASS: Appointment
Description: Represents scheduled visits between patients and doctors. This entity is essential for managing the timing and organization of patient care.
ATTRIBUTES:
- appointmentID: String
- patientID: String
- doctorID: String
- dateTime: Date
- status: String
METHODS:
- getAppointmentDetails(): Appointment
- reschedule(newDateTime: Date): void
- cancel(): void

CLASS: Visit
Description: Represents individual instances of patient consultations with healthcare providers. Each visit includes details such as diagnosis, treatment, and medications prescribed.
ATTRIBUTES:
- visitID: String
- patientID: String
- doctorID: String
- dateTime: Date
- diagnosis: Diagnosis
- treatment: Treatment
- medications: List<Medication>
METHODS:
- getVisitDetails(): Visit
- addDiagnosis(diagnosis: Diagnosis): void
- addTreatment(treatment: Treatment): void
- addMedication(medication: Medication): void

CLASS: Diagnosis
Description: Represents the identification of a medical condition based on patient visits. It is a key component of patient records and treatment plans.
ATTRIBUTES:
- diagnosisID: String
- description: String
- dateDiagnosed: Date
METHODS:
- getDiagnosisDetails(): Diagnosis
- updateDescription(newDescription: String): void

CLASS: Medication
Description: Represents prescribed drugs for patients. Multiple medications can be associated with a single visit, making this entity important for treatment tracking.
ATTRIBUTES:
- medicationID: String
- name: String
- dosage: String
- frequency: String
METHODS:
- getMedicationDetails(): Medication
- updateDosage(newDosage: String): void

CLASS: Treatment
Description: Represents the care provided to patients during visits, including procedures and therapies. It is essential for documenting patient care.
ATTRIBUTES:
- treatmentID: String
- description: String
- dateAdministered: Date
METHODS:
- getTreatmentDetails(): Treatment
- updateDescription(newDescription: String): void

CLASS: PatientHistory
Description: Represents the cumulative record of a patient's visits, diagnoses, and treatments over time. This is crucial for doctors to review prior to consultations.
ATTRIBUTES:
- patientID: String
- visits: List<Visit>
METHODS:
- getHistoryDetails(): PatientHistory
- addVisit(visit: Visit): void
